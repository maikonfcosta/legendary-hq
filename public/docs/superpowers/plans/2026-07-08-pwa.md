# PWA (Progressive Web App) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) ou superpowers:executing-plans para implementar este plano passo a passo. Utilize checkboxes (`- [ ]`) para monitorar o progresso.

**Goal:** Implementar a Prioridade 7 (PWA). Transformar o app em um Progressive Web App instalável, para que possa ser salvo na tela inicial do celular ou desktop e funcionar nativamente.

**Tech Stack:** `vite-plugin-pwa`

---

### Task 1: Instalação da Dependência

**Command:**
```bash
npm install vite-plugin-pwa -D
```

- [ ] Instalar o plugin `vite-plugin-pwa` como dependência de desenvolvimento.

---

### Task 2: Configuração do Vite

**Files:**
- Modificar: `vite.config.js`

- [ ] Adicionar e configurar a instância do `VitePWA` no array de plugins do Vite.
- [ ] O `VitePWA` deve registrar um Service Worker em modo 'autoUpdate', pre-cachear os assets e prover um `manifest` completo com o nome, descrição, cor de tema (#0f172a) e ícones corretos.

---

### Task 3: Gerar Ícones do PWA

- [ ] Usar a logo existente (`public/logo.jpg`) para configurar ícones PWA base que não deem erro ao ser carregados. Idealmente referenciando o `logo.jpg` em tamanhos `192x192` e `512x512` dentro do próprio manifesto, definindo-os como imagens suportadas, ou usar os ícones padrão caso já existam em public. (Obs: Se `logo.jpg` for usado, o "type" deve ser "image/jpeg").

---

### Task 4: Atualização da UI (Opcional, pois a UI do ReloadPrompt/Install já existe)

- [ ] A UI em `src/App.jsx` já possui escutas nativas para a instalação PWA (`beforeinstallprompt`). Só precisamos garantir que o Service Worker está sendo injetado pelo Vite.

---

### Task 5: Commit e Validação

```bash
git checkout -b feature/pwa preview
npm run lint
git add .
git commit -m "feat: setup PWA capabilities via vite-plugin-pwa"
```
